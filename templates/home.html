About Project

Because of the rapidly changing climate, the weather forecast is uncertain and inaccurate these days. As a result, the Weather Reporting System is primarily utilized to monitor the constantly changing climatic and weather conditions over-regulated areas like homes, industry, agriculture, and so on. The aim of our project is to monitor several aspects of weather using IoT based smart systems and to predict the future values.


Environmental Condition Monitoring System, an application capable enough to give you the predictions of whether it will rain or not in real-time based on a minimum number of parameters mainly current temperature and humidity allowing you to monitor real time weather and take the necessary steps.

The key features of the project are -

It uses the concept of edge-fog computing consisting of master and worker nodes, giving us the flexibility that if your worker is not able to process the request at the current moment, then master node will take over and will perform the necessary computations. You can access both master and worker by just simply logging into your account associated with the authorized worker.

It uses the concept of machine learning classification algorithms to give the predictions. The ML model is deployed on the web app which uses flask. Flask handles all the backend of the whole application and provides dynamic content to each website. Mongodb is also used for storing the necessary information for each user.

It uses real time data for making predictions which is made possible by using Raspberry Pi and DHT sensors, the data is sent to worker node (Raspberry Pi) for processing and the worker node makes the prediction and displays the results on web app (on local server) as the whole application is also accessible on the worker node itself.



It provides the user the flexibility to scale up the whole application. The user can connect multiple worker nodes and can manage them and allow all workers to work in complete isolation from each other i.e. separate data is stored for each and every worker and the results for the prediction are also stored separately and all the worker nodes are also accessible from the master.

All the predictions each and every worker makes are stored in google sheets with the help of google sheets api allowing us to view the past predictions and past inputs as temperature and humidity. Storing the results in the sheets also provides us with the option to update our model with real time data. The model can be trained from the predictions, thus allowing the model to be trained more efficiently and in short making the model more accurate.

All the information which is to be accessed by the user is secured through the login authentication system and also it is made sure that the results are displayed for valid users by using the IP Addresses of the worker nodes. It is ensured that only those workers can connect to the application and make computations which have valid IP Address and workerâ€™s IP address (which differs with different local networks mainly hotspots and wifi so it is important that you always connect you worker with the same local network) which is stored in the database at the time of registration. If both the IP Addresses match then only the predictions are accessible.

The main advantage of this project is that it helps in monitoring weather of a local area and thus it helps in developing a microclimate system. Based on the monitored database it predicts the future weather of a particular zone thereby making the result more accurate and relevant for the zone. This system is a portable alternative that can be adapted in broader applications. It is better than the existing websites which displays collective weather information whereas our system is more specific to the changes in weather adhering to a particular area.
